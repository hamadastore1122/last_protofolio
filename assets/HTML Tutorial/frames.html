By using frames you can display more than one page of your site in the same browser window.

Warning: Do not expect to continue to support frames in the future of HTML.

Frames

By using frames you can display more than one HTML page in the same browser window. Each HTML page is called a "frame" and each frame is independent of the others.

  • The disadvantages of using frames are:
  • Frames are not expected to be supported with HTML standalone.
  • Frames are difficult to use (especially if you print every page).
  • Web developers must keep track of each HTML page individually.

Frameset Elements

The main frame elements hold one or more frames, and each frame can hold an independent web page. The main frameset elements specify how many columns or rows will be in the main frame and how tall or wide each page will be displayed in pixels.

Tire Elements

The <frame> tag defines a frame page within the main frame. In the following example, we have a main frame with two columns.

<frameset cols="25%,75%">  
    <frame src="frame_a.html" />  
    <frame src="frame_b.html" />  
</frameset>

Note: The width of the frame columns can be adjusted using pixels, such as (cols="200,500"), and one of the columns can be adjusted by the remaining space by placing an asterisk (cols="25%,*").

Some Important Tips

  • If the frame borders are visible, the user can resize the frame by dragging the borders. To avoid resizing the frame and make the borders fixed, the noresize extension can be used for the <frame> tag, i.e., noresize="norezise".
  • Add <noframes> tag for browsers that do not support frames.
  • The body tags <body> cannot be used together when using <frameset> tags. However, if you add the <noframes> tag and it contains elements such as text specific to browsers that do not support frames, then the <body> tags must be closed.
