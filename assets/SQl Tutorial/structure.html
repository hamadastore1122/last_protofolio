SQL Expressions

Most of the commands you need to perform database operations are done through SQL expressions. In the following expression, all rows will be selected from the Persons table:

SELECT * FROM Persons

In this course, you will learn all different types of SQL expressions. Note that SQL is not case sensitive.

Comma after SQL Expressions

Some database systems require a semicolon after the end of each SQL expression. The semicolon is the standard way to end each database expression which allows more than one SQL expression to be used on the same server. The Arab Institute uses MySQL and sometimes it may not require a comma, but some programs require a comma after each expression.

What is DML and DDL?

SQL can be divided into two parts:

  • Data Manipulation Language (DML)
  • Data Definition Language (DDL)

DML is used with data update and modification commands:

  • SELECT: Select data from the database
  • UPDATE: Update data in the database
  • DELETE: Delete data from the database
  • INSERT INTO: Insert data into the database

DDL, a section of SQL, allows you to create or delete databases and tables, defines indexing, and defines the relationship between tables. The most important commands used with DDL are:

  • CREATE DATABASE: Create a database
  • ALTER DATABASE: Modify the database
  • CREATE TABLE: Create a table
  • ALTER TABLE: Modify the table
  • DROP TABLE: Delete the table
  • CREATE INDEX: Create an index or numbering for the table
  • DROP INDEX: Delete index
